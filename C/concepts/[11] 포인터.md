# Chapter 11 - 포인터 (Pointers)

## 포인터의 본질
포인터는 변수의 '값'이 아닌, 그 값이 저장된 '메모리 주소'를 저장하는 특수한 변수입니다. 자료구조에서 데이터 사이를 연결하는 '이정표' 역할을 함.

---

## 핵심 연산자
| 연산자 | 이름 | 설명 |
| :--- | :--- | :--- |
| `&` | 주소 연산자 | 변수가 메모리 어디에 있는지(주소)를 알아냄 |
| `*` | 참조(역참조) 연산자 | 포인터가 가리키는 주소로 직접 가서 값을 확인하거나 바꿈 |

```c
int num = 10;
int *ptr = &num; // num의 주소를 ptr에 저장

printf("num의 주소: %p\n", ptr);
printf("ptr이 가리키는 값: %d\n", *ptr); // 출력: 10
```

---

## 포인터와 배열
C언어에서 배열의 이름은 그 자체로 첫 번째 요소의 주소

- `arr` == `&arr[0]`
- 포인터에 1을 더하면 '1바이트'가 아니라 '가리키는 타입의 크기'만큼 주소가 이동
    - `int *p`에서 `p + 1`은 4바이트 이동

```c
int arr[3] = {10, 20, 30};
int *p = arr;

printf("%d\n", *(p + 1)); // 출력: 20 (arr[1]과 동일)
```

---

## 함수에서의 활용 (Call by Reference)
- 함수 외부의 데이터를 직접 수정하고 싶을 때 주소(포인터)를 넘겨줌. 
- 자료구조의 노드를 조작할 때 필수적인 방식입니다.

```c
void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main() {
    int x = 10, y = 20;
    swap(&x, &y); // 주소를 전달
    // x는 20, y는 10이 됨
}
```

---

## 자료구조와의 연결 (미리보기)
- 연결 리스트: `ptr->next`를 통해 다음 데이터 마디(Node)로 이동
- 트리: `ptr->left`, `ptr->right`를 통해 자식 데이터로 이동
- 동적 할당: `malloc`으로 만든 메모리 공간을 포인터로 관리

---

## 주의사항
1. NULL 초기화: 가리킬 주소가 없다면 반드시 `NULL`을 대입 (`int *p = NULL;`)
2. 잘못된 참조: 선언만 하고 주소를 넣지 않은 포인터를 사용하면 프로그램이 강제 종료

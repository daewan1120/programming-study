# Chapter 11 - 재귀함수 (Recursion)

## 재귀함수란?
- 함수 내부에서 자기 자신을 다시 호출하는 함수
- 큰 문제를 작은 단위의 동일한 문제로 쪼개어 해결하는 방식

---

## 재귀의 필수 조건
- 함수 내부에서 자기 자신을 다시 호출하는 함수
- 큰 문제를 작은 단위의 동일한 문제로 쪼개어 해결하는 방식

#### ① 탈출 조건 (Base Case)
언제 멈출 것인가? (가장 작은 문제 단위)
##### ex) if (n == 1) return 1;

#### ② 재귀 호출 (Recursive Case)

문제를 어떻게 쪼개서 자기 자신을 부를 것인가?
##### ex) return n * factorial(n - 1);

---

## 재귀의 필수 조건
재귀함수가 호출될 때마다 메모리의 스택 영역에는 함수의 복사본이 계속 쌓임

#### 호출 단계: main() -> f(3) -> f(2) -> f(1) 순으로 쌓임

#### 복귀 단계: 탈출 조건을 만나면 f(1) -> f(2) -> f(3) -> main() 순으로 결과값을 돌려주며 사라짐

---

## 예제: 팩토리얼(Factorial)

```c
#include <stdio.h>

int factorial(int n) {
    // 1. 탈출 조건
    if (n <= 1) {
        return 1;
    }
    // 2. 재귀 호출 (n! = n * (n-1)!)
    return n * factorial(n - 1);
}

int main() {
    int result = factorial(5); // 5 * 4 * 3 * 2 * 1
    printf("5! = %d", result);
    return 0;
}
```

## 재귀 vs 반복문

| 구분 | 재귀함수 (Recursion) | 반복문 (Iteration) |
|---|---|---|
| 코드 가독성 | 알고리즘이 직관적이고 코드가 간결함 | 코드가 길어질 수 있음 |
| 메모리 | 스택(Stack) 메모리 다량 사용 (Stack Overflow 위험) | 메모리 사용량이 적고 효율적임 |
| 속도 | 함수 호출 오버헤드로 인해 상대적으로 느림 | CPU 사이클을 직접 사용하여 상대적으로 빠름 |
| 적합한 상황 | 계층 구조(트리), 복잡한 탐색(DFS), 분할 정복 | 단순 반복 계산, 성능이 최우선인 경우 |
